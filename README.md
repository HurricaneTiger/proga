# MC LAN Tunnel

Играйте в Minecraft по локальной сети с друзьями через интернет. Это приложение создает туннель между игроками, позволяя подключаться к LAN-мирам без необходимости находиться в одной локальной сети.

## Возможности

- Создание комнат с уникальным кодом приглашения (6 символов)
- Подключение друзей по коду приглашения
- Автоматическое обнаружение порта Minecraft LAN
- Полноценный TCP-туннель через WebSocket-релей
- Темный игровой интерфейс на русском языке
- Автоматическое переподключение при разрыве связи
- Мониторинг состояния соединений в реальном времени
- Журнал событий с фильтрацией по уровню

## Требования

- Node.js 18+ (рекомендуется 20+)
- npm 9+
- Windows / macOS / Linux

## Запуск relay-сервера

Relay-сервер - это посредник, через который проходит весь трафик между хостом и клиентом.

```bash
# Скопировать пример конфигурации
cp apps/relay-server/.env.example apps/relay-server/.env

# Запустить сервер в режиме разработки
npm run server
```

Сервер запустится на порту 3000 (по умолчанию). Проверить работу можно по адресу: http://localhost:3000/health

### Переменные окружения (.env)

| Переменная | Описание | По умолчанию |
|---|---|---|
| `RELAY_PORT` | Порт HTTP/WebSocket сервера | `3000` |
| `WS_PATH` | Путь для WebSocket подключений | `/ws` |
| `MAX_ROOMS` | Максимальное количество комнат | `100` |
| `ROOM_TIMEOUT_MS` | Время жизни комнаты (мс) | `3600000` (1 час) |

## Как пользоваться: Хост (тот, кто открывает мир)

1. Запустите relay-сервер (или убедитесь, что он запущен на удаленном сервере)
2. Откройте десктопное приложение
3. Нажмите **"Создать комнату"**
4. Скопируйте код приглашения и отправьте его другу
5. Откройте мир в Minecraft и нажмите **"Открыть для сети"** (Open to LAN)
6. Запомните порт, который покажет Minecraft (или нажмите "Найти порт автоматически")
7. Введите порт в приложении
8. Нажмите **"Запустить туннель"**
9. Дождитесь подключения друга

## Как пользоваться: Клиент (тот, кто подключается)

1. Откройте десктопное приложение
2. Нажмите **"Подключиться к комнате"**
3. Введите код приглашения, полученный от хоста
4. Нажмите **"Подключиться"**
5. Скопируйте локальный адрес (например, `127.0.0.1:25565`)
6. В Minecraft: Мультиплеер -> Добавить сервер -> вставьте адрес
7. Подключитесь к серверу и играйте!

## Тестирование на одном компьютере

Для проверки работы можно запустить все на одной машине:

1. Запустите relay-сервер: `npm run server`
2. Запустите первый экземпляр приложения (хост)
3. Запустите второй экземпляр приложения (клиент)
4. В настройках обоих приложений адрес relay: `localhost:3000`
5. Создайте комнату в первом, подключитесь по коду во втором
6. Откройте Minecraft LAN-мир на хосте
7. Подключитесь по локальному адресу на клиенте

## Тестирование на двух компьютерах

1. Запустите relay-сервер на машине с публичным IP (или на VPS)
2. В `.env` файле relay-сервера порт должен быть открыт в файрволе
3. В настройках приложения у обоих игроков укажите IP relay-сервера (например: `192.168.1.100:3000` или `your-server.com:3000`)
4. Хост создает комнату, клиент подключается по коду
5. Далее стандартный процесс

## Необходимые порты

| Порт | Назначение | Где открыть |
|---|---|---|
| `3000` | Relay-сервер (HTTP + WebSocket) | На машине с relay-сервером |
| `25565` (или другой) | Minecraft LAN-порт | Только локально у хоста |
| `25565` | Локальный порт клиента для Minecraft | Только локально у клиента |

**Важно:** Хосту и клиенту не нужно открывать порты в файрволе/роутере - весь трафик идет через relay-сервер.

## Устранение неполадок

### Не удается подключиться к relay-серверу

- Убедитесь, что relay-сервер запущен: `curl http://localhost:3000/health`
- Проверьте адрес сервера в настройках приложения
- Убедитесь, что порт 3000 не занят другим приложением
- При удаленном подключении проверьте файрвол на сервере

### Код приглашения не работает

- Убедитесь, что код введен правильно (6 символов, буквы и цифры)
- Убедитесь, что хост все еще в комнате (код действует, пока хост онлайн)
- Проверьте, что оба подключены к одному relay-серверу

### Minecraft не видит сервер после подключения

- Убедитесь, что туннель запущен (индикатор зеленый)
- Проверьте, что в Minecraft открыт мир для LAN
- Убедитесь, что порт в приложении совпадает с портом в Minecraft
- Попробуйте добавить сервер вручную вместо поиска по LAN

### Высокий пинг / лаги

- Используйте relay-сервер географически ближе к обоим игрокам
- Проверьте скорость интернет-соединения у обоих игроков
- Закройте другие программы, использующие интернет

### "Хост отключился"

- Хост закрыл приложение или потерял соединение
- Попросите хоста перезапустить приложение и создать новую комнату

### Порт не определяется автоматически

- Убедитесь, что Minecraft запущен и мир открыт для LAN
- Проверьте расположение файла `latest.log` (стандартный путь: `%APPDATA%/.minecraft/logs/latest.log`)
- Введите порт вручную (он показывается в чате Minecraft при открытии мира для LAN)
